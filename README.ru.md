# VaultPad Server

[English version (README.md)](README.md) | [Репозиторий клиента](https://github.com/Terciuss/VaultPad)

Серверная часть [VaultPad](https://github.com/Terciuss/VaultPad) — зашифрованного хранилища данных с архитектурой нулевого знания. Сервер хранит и синхронизирует зашифрованные данные между клиентами. Он никогда не имеет доступа к открытым данным — всё шифрование и дешифрование выполняется на стороне клиента.

## Возможности

- **Хранение с нулевым знанием** — сервер хранит только зашифрованные имена и контент, открытые данные никогда не попадают на бэкенд
- **JWT-аутентификация** — токены HS256 со сроком действия 7 дней
- **Управление пользователями** — админ-панель для создания, редактирования и удаления пользователей
- **Доступ к файлам** — индивидуальное предоставление доступа к отдельным файлам
- **Автоматический сидер** — первый администратор создаётся автоматически при первом запуске (эндпоинт регистрации отсутствует)
- **Ограничение частоты** — 100 запросов на IP в минуту
- **Поддержка CORS** — настраиваемые кросс-доменные запросы
- **Docker** — multi-stage Dockerfile и docker-compose с MySQL

## Быстрый старт

### Docker Compose (рекомендуется)

```bash
cp .env.example .env
# Отредактируйте .env: задайте JWT_SECRET и MYSQL_ROOT_PASSWORD
docker compose up
```

Сервер запустится на порту `8080` вместе с MySQL. Первый администратор создаётся автоматически с email из `SEED_ADMIN_EMAIL`.

### Локальная разработка

```bash
cp .env.example .env
# Отредактируйте .env: замените "db" на "127.0.0.1" в DATABASE_DSN
# Убедитесь, что MySQL запущен локально

go run ./cmd/server
```

Требования: Go 1.25+, MySQL 8+

## Переменные окружения

| Переменная | По умолчанию | Описание |
|---|---|---|
| `DATABASE_DSN` | — | Строка подключения MySQL (например `root:password@tcp(127.0.0.1:3306)/vault_pad?parseTime=true`) |
| `JWT_SECRET` | — | Секретный ключ для подписи JWT (рекомендуется минимум 32 символа) |
| `LISTEN_ADDR` | `:8080` | Адрес и порт сервера |
| `SEED_ADMIN_EMAIL` | `admin@local.local` | Email для автоматически создаваемого администратора |
| `MYSQL_ROOT_PASSWORD` | `root` | Пароль root MySQL (только для Docker) |
| `MYSQL_DATABASE` | `vault_pad` | Имя базы данных MySQL (только для Docker) |

## API-эндпоинты

### Публичные

| Метод | Путь | Описание |
|---|---|---|
| `POST` | `/api/auth/login` | Аутентификация, получение JWT-токена |
| `GET` | `/api/health` | Проверка состояния |

### С аутентификацией (требуется JWT)

| Метод | Путь | Описание |
|---|---|---|
| `GET` | `/api/auth/me` | Информация о текущем пользователе |
| `GET` | `/api/projects/meta` | Метаданные проектов (ID, временные метки) для синхронизации |
| `GET` | `/api/projects` | Список доступных проектов |
| `GET` | `/api/projects/{id}` | Получить проект по ID |
| `POST` | `/api/projects` | Создать проект |
| `PUT` | `/api/projects/{id}` | Обновить проект |
| `DELETE` | `/api/projects/{id}` | Удалить проект (только владелец/админ) |

### Админские (JWT + роль администратора)

| Метод | Путь | Описание |
|---|---|---|
| `GET` | `/api/admin/users` | Список всех пользователей |
| `POST` | `/api/admin/users` | Создать пользователя |
| `PUT` | `/api/admin/users/{id}` | Обновить пользователя |
| `DELETE` | `/api/admin/users/{id}` | Удалить пользователя (нельзя удалить последнего админа) |
| `GET` | `/api/admin/users/{id}/shares` | Список расшаренных проектов пользователя |
| `POST` | `/api/projects/{id}/share` | Предоставить доступ к проекту |
| `DELETE` | `/api/projects/{id}/share/{userId}` | Отозвать доступ к проекту |

## Схема базы данных

### users

| Колонка | Тип | Описание |
|---|---|---|
| `id` | BIGINT PK | Автоинкрементный ID |
| `email` | VARCHAR(255) UNIQUE | Email пользователя |
| `password_hash` | VARCHAR(255) | bcrypt-хеш пароля |
| `is_admin` | BOOLEAN | Флаг администратора |
| `created_at` | TIMESTAMP | Дата создания |

### projects

| Колонка | Тип | Описание |
|---|---|---|
| `id` | BIGINT PK | Автоинкрементный ID |
| `user_id` | BIGINT FK | Владелец |
| `encrypted_name` | BLOB | Зашифрованное имя файла |
| `encrypted_content` | MEDIUMBLOB | Зашифрованный контент файла |
| `sort_order` | INT | Порядок отображения |
| `created_at` | TIMESTAMP | Дата создания |
| `updated_at` | TIMESTAMP | Дата последнего обновления |

### project_shares

| Колонка | Тип | Описание |
|---|---|---|
| `id` | BIGINT PK | Автоинкрементный ID |
| `project_id` | BIGINT FK | Расшаренный проект |
| `user_id` | BIGINT FK | Пользователь-получатель |
| `shared_by` | BIGINT | Кто расшарил |
| `created_at` | TIMESTAMP | Дата расшаривания |

## Контроль доступа

- **Владелец** — полный доступ к своим проектам
- **Администратор** — полный доступ ко всем проектам
- **Пользователь с доступом** — чтение и обновление расшаренных проектов (без удаления)
- Последнего администратора нельзя удалить или понизить

## Безопасность

- Все данные на сервере зашифрованы на стороне клиента (AES-256-GCM)
- Пароли хешируются bcrypt
- JWT-токены истекают через 7 дней
- Ограничение частоты: 100 запросов/мин на IP
- Эндпоинт регистрации отсутствует — пользователи создаются администраторами или через автосидер
